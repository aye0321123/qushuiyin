# AI改写功能诊断指南

## 问题描述
用户在使用AI改写功能时遇到"请求异常，让我检查网络"错误。

## 诊断步骤

### 1. 使用测试连接功能
1. 打开网页后，解析一个视频
2. 在AI改写区域，点击 **🔧 测试连接** 按钮
3. 查看弹出的提示信息，会显示详细的错误原因

### 2. 检查浏览器控制台
1. 按 **F12** 打开浏览器开发者工具
2. 切换到 **Console（控制台）** 标签
3. 点击"测试连接"或"改写"按钮
4. 查看控制台中的错误信息（红色文字）

### 3. 常见问题及解决方案

#### 问题1: 混合内容（Mixed Content）
**症状**: 错误信息包含 "Failed to fetch" 或 "Mixed Content"

**原因**: 
- 如果网页使用 HTTPS 访问，但 AI API 使用 HTTP
- 浏览器会阻止 HTTPS 页面访问 HTTP 资源（安全策略）

**解决方案**:
- 使用 HTTP 协议访问网页（例如: `http://localhost/index.html`）
- 或者将 AI API 升级为 HTTPS

**检查方法**:
```
当前页面协议: 查看浏览器地址栏
AI API 地址: http://43.139.203.146:8050
```

#### 问题2: API服务器不可用
**症状**: 错误信息包含 "NetworkError" 或 "Failed to fetch"

**原因**:
- API 服务器 (43.139.203.146:8050) 已关闭
- IP 地址或端口已更改
- 服务器防火墙阻止访问

**解决方案**:
1. 在浏览器中直接访问: `http://43.139.203.146:8050/v1/chat/completions`
2. 如果无法访问，联系管理员确认服务器状态
3. 检查服务器是否正常运行

#### 问题3: CORS跨域限制
**症状**: 控制台显示 "CORS policy" 相关错误

**原因**: API 服务器未配置允许跨域访问

**解决方案**:
- 需要在 API 服务器端配置 CORS 响应头
- 添加以下响应头:
  ```
  Access-Control-Allow-Origin: *
  Access-Control-Allow-Methods: POST, OPTIONS
  Access-Control-Allow-Headers: Content-Type, Authorization
  ```

#### 问题4: API密钥无效
**症状**: HTTP 401 或 403 错误

**当前配置**:
```javascript
API地址: http://43.139.203.146:8050/v1/chat/completions
API密钥: sk-123456
模型: deepseek-chat
```

**解决方案**:
- 联系管理员确认密钥是否有效
- 检查密钥是否过期或被禁用

#### 问题5: 防火墙或网络限制
**症状**: 连接超时或无响应

**解决方案**:
- 检查本地防火墙设置
- 尝试关闭 VPN 或代理
- 检查公司/学校网络是否有限制
- 尝试使用其他网络（如手机热点）

### 4. 测试API连接的其他方法

#### 使用 curl 命令测试（Windows PowerShell）:
```powershell
curl -X POST "http://43.139.203.146:8050/v1/chat/completions" `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer sk-123456" `
  -d '{
    "model": "deepseek-chat",
    "messages": [
      {"role": "system", "content": "你是一个AI助手"},
      {"role": "user", "content": "测试连接"}
    ],
    "temperature": 0.7,
    "max_tokens": 50
  }'
```

#### 使用浏览器直接访问:
在浏览器地址栏输入:
```
http://43.139.203.146:8050
```
查看是否能访问服务器

### 5. 临时解决方案

如果无法解决连接问题，可以考虑:

1. **更换API服务器**: 使用其他可用的 DeepSeek API 服务
2. **使用代理**: 通过后端代理转发请求
3. **本地部署**: 在本地搭建 AI API 服务
