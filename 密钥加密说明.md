# 🔐 密钥加密功能说明

## ✨ 功能介绍

为了保护 API 密钥不被轻易从源码中获取，我们添加了密钥加密功能。

## 🔒 加密原理

使用 **Base64 编码 + XOR 异或加密** 的组合方式：
1. 对原始密钥的每个字符进行 XOR 异或运算（密钥：42）
2. 将结果进行 Base64 编码
3. 存储加密后的字符串

## 🛠️ 如何更换密钥

### 方法一：使用加密工具（推荐）

1. 打开 `密钥加密工具.html`
2. 在"原始密钥"输入框输入新密钥
3. 点击"🔒 加密密钥"按钮
4. 复制加密后的密钥
5. 替换 `index.html` 中对应的加密密钥

### 方法二：手动加密

使用以下 JavaScript 代码：

```javascript
function encrypt(text) {
  let encrypted = '';
  for (let i = 0; i < text.length; i++) {
    encrypted += String.fromCharCode(text.charCodeAt(i) ^ 42);
  }
  return btoa(encrypted);
}

// 使用示例
const newKey = 'your-new-api-key';
const encrypted = encrypt(newKey);
console.log(encrypted);
```

## 🔓 解密过程

在 `index.html` 中，密钥会在运行时自动解密：

function decryptKey(encrypted) {
  const decoded = atob(encrypted);
  let result = '';
  for (let i = 0; i < decoded.length; i++) {
    result += String.fromCharCode(decoded.charCodeAt(i) ^ 42);
  }
  return result;
}

const GLOBAL_API_KEY = decryptKey(ENCRYPTED_GLOBAL_KEY);

## 🎯 安全级别

### 当前方案（已实现）
- **安全级别**: ⭐⭐⭐ 中等
- **优点**: 
  - 防止直接查看源码获取密钥
  - 增加了获取难度
  - 不影响性能
- **缺点**: 
  - 仍可通过控制台调试获取
  - 加密算法相对简单

### 更高安全方案（可选）
如需更高安全性，可以考虑：
1. **后端代理**: 将 API 密钥存储在后端，前端通过后端接口调用
2. **环境变量**: 使用构建工具（如 Webpack）在编译时注入密钥
3. **动态密钥**: 实现密钥轮换机制

## 📊 对比

| 方案 | 安全性 | 实现难度 | 性能影响 |
|------|--------|----------|----------|
| 明文存储 | ⭐ | 简单 | 无 |
| Base64+XOR（当前） | ⭐⭐⭐ | 简单 | 极小 |
| 后端代理 | ⭐⭐⭐⭐⭐ | 复杂 | 有 |

## ⚠️ 注意事项

1. **不要泄露加密工具**: `密钥加密工具.html` 不要部署到公开网站
2. **定期更换密钥**: 建议每月更换一次 API 密钥
3. **监控使用情况**: 定期检查 API 调用日志，发现异常及时更换密钥
4. **备份密钥**: 保存原始密钥的备份，避免丢失
