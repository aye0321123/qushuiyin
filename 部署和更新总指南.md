## 🌐 部署到 GitHub Pages

### 前提条件

- ✅ GitHub 账号：`aye0321123`
- ✅ 已创建仓库（如果还没有，请先创建）

### 部署步骤

#### 方法一：网页上传（最简单）

1. **访问您的 GitHub 仓库**
   ```
   https://github.com/aye0321123/仓库名
   ```

2. **上传文件**
   - 点击 **Add file** → **Upload files**
   - 拖拽以下文件：
     - `index.html`
     - `admin.html`
     - `bg.jpg`
   - 点击 **Commit changes**

3. **启用 GitHub Pages**
   - 点击仓库的 **Settings**
   - 左侧菜单找到 **Pages**
   - Branch 选择：**main**
   - Folder 选择：**/ (root)**
   - 点击 **Save**

4. **等待部署**
   - 等待 1-2 分钟
   - 访问：`https://aye0321123.github.io/仓库名/`

#### 方法二：使用 Git（推荐，适合频繁更新）

1. **安装 Git**
   - 参考：`安装Git指南.md`
   - 下载：https://git-scm.com/download/win

2. **配置 Git**
   ```bash
   git config --global user.name "aye0321123"
   git config --global user.email "你的邮箱@example.com"
   ```

3. **使用自动上传脚本**
   - 双击：`自动上传到GitHub.bat`
   - 首次使用输入仓库地址
   - 输入用户名和 Token
   - 等待上传完成

4. **启用 GitHub Pages**
   - 同方法一的步骤 3

## 🔧 常见问题

### Q1: 我应该选择哪种更新方式？

**回答**：
- 如果您**不想安装软件**，选择**网页上传**
- 如果您**经常更新**，选择**Git 自动上传**
- 如果您**不确定**，先用**网页上传**试试

### Q2: Git 安装失败怎么办？

**回答**：
1. 使用网页上传方式（不需要 Git）
2. 查看 `安装Git指南.md` 的常见问题部分
3. 对 Kiro 说："Git 安装失败了，帮我看看"

### Q3: 上传后网站没有更新？

**回答**：
1. 等待 2-3 分钟（GitHub Pages 需要时间）
2. 按 `Ctrl + F5` 强制刷新浏览器
3. 检查 GitHub Actions 是否成功
4. 清除浏览器缓存

### Q4: 如何查看网站是否部署成功？

**回答**：
1. 访问仓库的 **Settings** → **Pages**
2. 看到绿色提示：`Your site is live at https://...`
3. 点击链接访问网站

### Q5: 忘记 GitHub 密码怎么办？

**回答**：
- 使用 **Personal Access Token**，不是 GitHub 密码
- Token 获取：https://github.com/settings/tokens
- 详细步骤见 `安装Git指南.md`

### Q6: 可以在手机上更新吗？

**回答**：
- ✅ 可以！使用网页上传方式
- 在手机浏览器访问 GitHub
- 编辑文件并提交
- 但建议使用电脑操作（更方便）

### Q7: 如何撤销错误的更新？

**回答**：
1. 访问文件的 **History**
2. 找到正确的版本
3. 复制内容
4. 重新编辑文件并提交

### Q8: 更新后出现错误怎么办？

**回答**：
1. 对 Kiro 说："网站出错了，帮我看看"
2. 提供错误截图或描述
3. Kiro 会帮你修复
