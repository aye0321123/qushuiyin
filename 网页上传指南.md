# 🌐 GitHub 网页上传指南（无需安装 Git）

## 📋 适用场景

如果您：
- ❌ 不想安装 Git
- ❌ 只需要偶尔更新
- ✅ 想要最简单的方式

那么这个方法最适合您！

---

## 🚀 方法一：直接编辑文件（推荐）

### 适用于：修改现有文件（如 index.html）

#### 步骤：

1. **打开您的 GitHub 仓库**
   ```
   https://github.com/aye0321123/仓库名
   ```

2. **找到要修改的文件**
   - 点击文件名（例如：`index.html`）

3. **点击编辑按钮**
   - 点击右上角的 **铅笔图标** 📝（Edit this file）

4. **修改内容**
   - 删除旧内容
   - 粘贴新内容（从本地文件复制）

5. **提交更改**
   - 滚动到页面底部
   - 在 "Commit message" 输入：`更新视频解析工具`
   - 点击绿色按钮 **Commit changes**

6. **等待部署**
   - 等待 1-2 分钟
   - 访问您的网站查看更新

## 📤 方法二：上传新文件

### 适用于：添加新文件或替换多个文件

#### 步骤：

1. **打开您的 GitHub 仓库**
   ```
   https://github.com/aye0321123/仓库名
   ```

2. **点击上传按钮**
   - 点击 **Add file** → **Upload files**

3. **选择文件**
   - 拖拽文件到页面
   - 或点击 "choose your files" 选择文件

4. **提交更改**
   - 在 "Commit message" 输入：`更新视频解析工具`
   - 点击 **Commit changes**

5. **等待部署**
   - 等待 1-2 分钟
   - 访问您的网站查看更新

## 🔄 完整更新流程示例

### 场景：Kiro 帮你修改了 index.html

#### 步骤 1：复制新内容

1. 在本地打开 `index.html`（右键 → 打开方式 → 记事本）
2. 按 `Ctrl + A` 全选
3. 按 `Ctrl + C` 复制

1. 访问：`https://github.com/aye0321123/仓库名`
2. 点击 `index.html`
3. 点击右上角 **铅笔图标** 📝

#### 步骤 3：替换内容

1. 在编辑器中按 `Ctrl + A` 全选
2. 按 `Ctrl + V` 粘贴新内容

1. 滚动到底部
2. 输入提交信息：`更新视频解析工具`
3. 点击 **Commit changes**

#### 步骤 5：验证

1. 等待 1-2 分钟
2. 访问：`https://aye0321123.github.io/仓库名/`
3. 按 `Ctrl + F5` 强制刷新页面
4. 查看更新是否生效

## 🔧 常见问题

### Q1: 提交后网站没有更新？

**解决方法**：
1. 等待 2-3 分钟（GitHub Pages 需要时间部署）
2. 按 `Ctrl + F5` 强制刷新浏览器
3. 清除浏览器缓存
4. 检查 GitHub Actions 是否成功（仓库 → Actions 标签）

### Q2: 编辑时提示 "You're editing a file in a project you don't have write access to"

**原因**：您不是仓库的所有者或没有写入权限

**解决方法**：
1. 确认您已登录 GitHub
2. 确认您是仓库的所有者
3. 如果是协作者，需要仓库所有者授予权限

### Q3: 如何查看更新历史？

**方法**：
1. 打开仓库主页
2. 点击右上角的 **Clock 图标**（Commits）
3. 查看所有提交记录

### Q4: 如何撤销错误的更新？

**方法**：
1. 打开文件
2. 点击 **History** 按钮
3. 找到正确的版本
4. 点击 **<>** 图标查看内容
5. 复制内容后重新编辑文件

### Q5: 可以同时更新多个文件吗？

**可以！使用上传方式**：
1. 点击 **Add file** → **Upload files**
2. 拖拽多个文件到页面
3. 一次性提交

## 💡 最佳实践

### 1. 提交信息要清晰

❌ 不好的提交信息：
- "更新"
- "修改"
- "fix"

✅ 好的提交信息：
- "修复下载按钮样式问题"
- "添加 AI 改写功能"
- "更新平台图标"

### 2. 更新前先备份

在修改前：
1. 复制原文件内容到记事本
2. 保存为 `index.html.backup`
3. 如果出错可以快速恢复

### 3. 分批更新

如果有多个修改：
1. 先更新主要功能
2. 测试是否正常
3. 再更新次要功能

### 4. 使用浏览器书签

将以下链接添加到书签：
```
https://github.com/aye0321123/仓库名/edit/main/index.html
```

下次直接点击书签即可编辑！
