## 🚀 上传步骤（详细图文）

### 步骤 1：打开 GitHub 仓库

1. 打开浏览器
2. 访问：https://github.com/aye0321123/qushuiyin
3. 登录您的 GitHub 账号

---

### 步骤 2：进入 qushuiyin 文件夹

1. 在仓库主页，点击 **qushuiyin** 文件夹
2. 找到 **index.html** 文件

---

### 步骤 3：编辑 index.html

1. 点击 **index.html** 文件
2. 点击右上角的 **铅笔图标** 📝（Edit this file）
3. 会进入编辑模式

---

### 步骤 4：替换内容

1. **在编辑器中按 Ctrl + A 全选所有内容**
2. **按 Delete 删除**
3. **打开本地文件**：
   - 按 Win + E 打开文件管理器
   - 导航到：`C:\Users\Administrator.SY-202410161910\qushuiyin`
   - 右键点击 `index.html`
   - 选择"打开方式" → "记事本"
4. **在记事本中按 Ctrl + A 全选**
5. **按 Ctrl + C 复制**
6. **回到 GitHub 编辑页面**
7. **按 Ctrl + V 粘贴**

---

### 步骤 5：提交更改

1. 滚动到页面底部
2. 在 **"Commit changes"** 输入框中输入：
   ```
   优化移动端显示效果
   ```
3. 可选：在下方大文本框输入详细说明：
   ```
   - 优化触摸体验
   - 改进按钮大小
   - 优化文字和间距
   - 侧边栏改为横向滚动
   - 视频信息单列显示
   ```
4. 点击绿色按钮 **"Commit changes"**

---

### 步骤 6：等待部署

1. 提交后会自动返回文件列表
2. 点击顶部的 **"Actions"** 标签
3. 查看部署进度（会显示黄色圆圈 🟡 表示正在部署）
4. 等待变成绿色勾号 ✅（大约 1-2 分钟）

---

### 步骤 7：验证更新

1. 等待部署完成后
2. 在手机浏览器访问：
   ```
   https://aye0321123.github.io/qushuiyin/qushuiyin/
   ```
3. **强制刷新**：
   - iOS Safari：下拉刷新
   - Android Chrome：下拉刷新
   - 或清除缓存后重新访问

## ⚠️ 注意事项

### 1. 确保完整复制
- ✅ 必须复制整个文件内容
- ✅ 不要遗漏开头和结尾
- ✅ 检查是否有 `<!DOCTYPE html>` 开头
- ✅ 检查是否有 `</html>` 结尾

### 2. 提交信息
- ✅ 简短描述更改内容
- ✅ 使用中文或英文都可以
- ✅ 避免特殊字符

### 3. 等待部署
- ⏱️ 通常需要 1-2 分钟
- ⏱️ 不要在部署期间再次提交
- ⏱️ 可以在 Actions 页面查看进度

### 4. 清除缓存
- 📱 手机浏览器可能会缓存旧版本
- 📱 需要强制刷新或清除缓存
- 📱 或使用无痕模式访问

## 🔧 常见问题

### Q1: 编辑按钮找不到？
**A**: 确保您已登录 GitHub，且在正确的文件页面

### Q2: 提交后没有变化？
**A**: 
1. 等待 2-3 分钟
2. 清除浏览器缓存
3. 使用无痕模式访问
4. 检查 Actions 是否部署成功

### Q3: 显示 404 错误？
**A**: 
1. 检查网址是否正确
2. 确认文件已成功上传
3. 等待几分钟后重试

### Q4: 复制内容时出错？
**A**: 
1. 确保使用记事本打开（不要用 Word）
2. 全选后再复制
3. 粘贴到 GitHub 后检查格式
